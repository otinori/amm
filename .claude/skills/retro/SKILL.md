# /retro — 振り返り (Retrospective)

作業中に `.udr/retro-pending.md` へ蓄積した問題リストをもとに対策を設計し、
AGENTS.md / CLAUDE.md / skills に反映する。
「何があったか」のヒアリングは行わない——問題は既に蓄積済みのはず。

---

## 実行手順

### Step 1: 蓄積リストを読む

```bash
cat .udr/retro-pending.md 2>/dev/null || echo "(エントリなし)"
```

エントリがない場合: 「`.udr/retro-pending.md` にエントリがありません。今回は特に問題なく進みましたか？」と確認し、ユーザーが追加したい場合は口頭で受け取って以降の手順へ進む。

---

### Step 2: カテゴリ分類

各エントリを以下のカテゴリに分類する（AI が判断、ユーザーに確認は不要）:

| カテゴリ | 説明 | 主な改善場所 |
|---|---|---|
| **knowledge** | ドメイン知識・仕様の理解不足 | `AGENTS.md` に知識・リファレンスを追記 |
| **pattern** | 既存パターンの見落とし | `AGENTS.md` にチェックリスト追加 |
| **sideeffect** | 副作用・波及効果の告知漏れ | `AGENTS.md` に副作用表を追記 |
| **scope** | 作業スコープ・意図の誤解 | `CLAUDE.md` に運用ルール追加 |
| **tooling** | 繰り返し作業の skill 化が必要 | `.claude/skills/<name>/SKILL.md` を新規作成 |
| **process** | フェーズ遷移ルールの不明確さ | `AGENTS.md §6` の開発ループを更新 |

---

### Step 3: 対策案の設計と提示

各エントリに対して対策案を生成し、追記内容のプレビューをまとめて表示する:

```
## /retro 対策案

### 1. <問題の一行説明>
- カテゴリ: <category>
- 改善場所: <ファイル + セクション>
- 追記内容:
  ────────────────────
  （追記予定のテキストをそのまま表示）
  ────────────────────

### 2. ...

---
上記を反映します。個別にスキップしたい項目があれば番号で指定してください。
なければそのまま Enter で全件反映します。
```

ユーザーの確認を1回だけ取る（項目ごとに都度確認しない）。

---

### Step 4: 承認された対策の反映

スキップ指定がなければ全件、指定があればそれ以外を反映する:

1. 対象ファイルを Read で確認し、適切な挿入位置を特定
2. Edit で追記
3. 反映後に「〇〇に追記しました」と報告

---

### Step 5: retro-pending.md をクリアしてコミット

```bash
echo "" > .udr/retro-pending.md
```

反映した改善を 1件1コミットで `retro:` プレフィックスでコミットする:

```bash
git add <変更ファイル>
git commit -m "retro: <改善内容の一行サマリ>"
```

全件コミット完了後:

```bash
git add .udr/retro-pending.md
git commit -m "retro: clear retro-pending after retrospective"
```

---

### Step 6: 完了報告

```
## /retro 完了

反映した改善 (N件):
- retro: <コミット1>
- retro: <コミット2>
...

スキップした問題:
- <問題名>（理由: ユーザー指定）

次のセッションからこれらのガードレールが有効です。
main への最終 PR を作成しますか？
```
